# 减重趋势分析

## 概述

本技能用于整理用户已经提供的体重、饮食、活动、睡眠和围度记录，计算可核验的描述性趋势，并帮助用户准备与医生或注册营养师讨论的问题。它不诊断疾病，不判定“理想体重”，也不生成个体化热量缺口、快速减重、禁食或运动处方。

## When to Use

- 用户希望汇总一段时间内的体重或围度记录。
- 用户希望识别记录完整性、短期波动和长期趋势。
- 用户需要比较自己设定的目标与实际记录，但没有要求系统制定医疗或营养方案。
- 用户希望把数据整理成可供专业人员审阅的报告。

## 不适用范围

- 诊断肥胖、营养不良、进食障碍、代谢疾病或其他健康状况。
- 根据性别、BMI、体脂率或公式自动开出摄入量、缺口、目标体重或减重速度。
- 给出快速减重、间歇性禁食、脱水、补剂或高强度运动方案。
- 为未成年人、孕期或哺乳期人群制定减重计划。

上述请求应转介具备相应资质的医生或注册营养师。出现晕厥、胸痛、呼吸困难、严重虚弱、持续呕吐、意识改变或其他急性症状时，建议立即联系当地急救服务。

## 工作流程

### 1. 确认数据与目的

只使用用户明确提供或授权读取的数据。先确认：

- 日期、体重及单位；
- 测量时间和条件是否大致一致；
- 可选的围度、睡眠、活动或饮食记录；
- 用户自己设定的目标和希望回答的问题；
- 缺失值、设备变更、旅行、疾病或其他可能影响记录的事件。

不要声称存在本技能包未提供的追踪器、数据文件或斜杠命令。

### 2. 规范化记录

- 保留原始值，并在副本中统一单位。
- 不推测缺失日期、摄入量、运动消耗或身体成分。
- 标注设备、测量条件或数据来源的变化。
- 对明显录入错误先向用户确认，不擅自删除。

### 3. 描述趋势

可报告以下可复核指标：

- 起止值和净变化；
- 每周或每月的观测次数；
- 7 日或用户指定窗口的移动平均；
- 周与周之间的平均值变化；
- 数据缺口和异常波动。

短期体重可能受水分、盐分、糖原、消化内容物和测量条件影响。除非有临床评估，不把单次变化解释为脂肪增减或代谢改变。

如果用户要求 BMI，可按 `体重(kg) / 身高(m)^2` 计算并清楚标注：它是群体筛查指标，不是诊断，也不能单独确定健康状态、体脂或个体目标。不要自动添加分类标签或目标体重。

### 4. 处理热量与活动数据

- 把设备的活动消耗和公式估算标记为估计值，不当作精确测量。
- 不把已包含活动水平的 TDEE 估计与单独运动消耗再次相加。
- 不使用固定“每公斤脂肪对应热量”规则预测个人减重时间。
- 不从 BMR/TDEE 公式推导个体化摄入目标或“安全最低热量”。
- 若用户已有专业人员制定的计划，只比较记录与该计划，不自行调整处方。

### 5. 输出与下一步

区分事实、计算和不确定性：

```markdown
# 体重趋势报告

## 数据范围
- 期间：2026-06-01 至 2026-06-28
- 有效记录：22 / 28 天
- 单位与条件：kg；多数为晨起测量

## 描述性结果
- 起始记录：用户提供值
- 最新记录：用户提供值
- 净变化：最新值减起始值
- 7 日移动平均：列出每周末的计算结果

## 数据质量与不确定性
- 缺失日期：列出日期
- 条件变化：设备或测量时间变化
- 不能从这些数据判断：脂肪变化、代谢状态或疾病原因

## 可讨论的问题
- 这个趋势是否符合用户与专业人员已经确定的目标？
- 是否需要由医生或注册营养师审阅药物、症状或计划？
```

若用户希望改变饮食、运动或体重目标，提供低风险的下一步：保存一致的记录、整理问题、预约具备资质的专业人员。不要把通用资料转化为个人处方。

## 安全与隐私

- 健康记录属于敏感数据；只读取必要范围，不在日志中复制完整记录。
- 不根据姓名、照片或外观推断体重、性别、疾病或进食行为。
- 如果用户提到催吐、滥用泻药、极端限制、强迫运动、明显进食恐惧或自伤想法，以支持性语言建议尽快联系专业人员或当地危机/急救资源，不继续优化减重计划。
- 有慢性病、处方药、近期手术、进食障碍史、妊娠/哺乳或不明症状时，个体化计划必须由专业人员评估。

## Limitations

- 本技能仅提供描述性数据整理，不构成医疗、营养或运动建议。
- BMR、TDEE、穿戴设备消耗和体脂估计都可能有显著误差。
- 无法从体重趋势单独判断脂肪、肌肉、水分或健康风险的变化。
- 缺少必要数据、授权或安全边界时，停止并向用户澄清。
