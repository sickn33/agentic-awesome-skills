# 性健康记录分析

## When to Use

- 用户希望整理自己提供的检测、症状、避孕或用药记录。
- 用户需要为医生、性健康门诊或其他合格专业人员准备时间线和问题清单。
- 用户希望比较一段时间内的记录完整性、检测随访或自定目标。

## Safety, Consent, and Privacy Boundary

本技能只进行信息整理，不进行以下操作：

- 不诊断性传播感染、勃起功能障碍、不孕症或其他疾病。
- 不使用自建分数按身份、职业、性取向或伴侣数量给个人贴风险标签。
- 不推荐或调整抗生素、抗病毒药、激素、补充剂、避孕药或其他治疗及剂量。
- 不替代检测窗口、筛查频率、避孕选择或妊娠风险的个体化临床判断。
- 不假定任何性行为是自愿的，也不要求与任务无关的敏感细节。

如果用户描述强迫、暴力、性侵、无法安全表达同意、立即危险或严重急性症状，停止常规分析并优先建议联系当地紧急服务、专业医疗机构或可信的反暴力支持资源。近期可能暴露于 HIV、其他感染或意外妊娠可能涉及时间敏感的医疗选择；建议尽快联系当地急诊或性健康专业人员，而不是由本技能决定措施。

涉及未成年人时，不开展成人式性行为评估；遵循适用的保护、隐私和强制报告规则，并转介合格的医疗或保护专业人员。

## Data Minimization

只收集完成任务所需且由用户自愿提供的信息：

- 日期、检测类型、结果来源和专业人员给出的随访计划。
- 用户原文描述的症状、持续时间和变化。
- 用户希望讨论的避孕、妊娠、用药或功能方面的问题。
- 用户明确同意纳入的伴侣沟通或暴露时间线。

不要索取姓名、地址、证件、伴侣身份或可避免的详细性行为描述。输出中去标识化，并区分“用户报告”“实验室结果”和“专业人员建议”。

## Workflow

1. **确认目标与同意**：说明将整理哪些敏感信息，以及用户可跳过任何问题。
2. **检查紧急情况**：先应用安全、同意和隐私边界。
3. **建立时间线**：按日期列出检测、症状、用药变化和专业随访。
4. **标记缺口**：指出缺少的日期、检测类型、结果来源或随访状态，不推断结果。
5. **避免身份代理**：只描述具体、用户自愿提供的事件，不以群体身份替代个体评估。
6. **准备专业咨询**：生成可向临床人员提出的问题，并引用当前官方资源供核对。
7. **校验输出**：确认没有诊断、风险百分比、治疗建议、虚构经历或不必要的敏感数据。

## Evidence Rules

- 不把症状与某一感染或疾病直接对应。
- 不把检测阴性解释为所有感染均已排除；检测时间和类型由临床人员确认。
- 不创建“低、中、高风险”分数或基于身份的权重。
- 不编造伴侣、行为、实验室值、医疗史或同意情况。
- 医疗建议必须明确归属于用户提供的专业记录或当前官方来源。

## Limitations

- 本技能不能替代医疗检测、临床评估、危机支持或法律与保护服务。
- 不提供诊断、处方、剂量、治疗方案或个体化风险结论。
- 对检测时间、避孕、妊娠、用药或症状的决定应由合格专业人员结合当地指南完成。
