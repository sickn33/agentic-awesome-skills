# 口腔健康记录分析

## When to Use

- 用户希望整理刷牙、牙线、洁牙、症状或牙科就诊记录。
- 用户需要比较一段时间内的变化，或为牙科就诊准备问题清单。
- 用户需要把口腔记录与自己提供的饮食、吸烟、用药或慢性病信息按时间关联。

## Safety Boundary

本技能只整理用户提供的信息，不进行以下操作：

- 不诊断龋齿、牙周病、感染、口腔癌或其他疾病。
- 不依据自建分数进行疾病分期、预后或个体风险预测。
- 不推荐处方药、抗生素、补充剂、剂量或治疗方案。
- 不判断药物、妊娠或慢性病情况下的牙科治疗是否安全。
- 不把缺少症状、照片或历史记录解释为“没有问题”。

出现呼吸或吞咽困难、快速加重的面部或口腔肿胀、无法控制的出血、严重外伤，或伴全身不适的急性症状时，停止常规分析并建议立即联系当地急救服务或紧急牙科医疗。持续不愈的口腔溃疡、肿块、白斑或红斑等问题应尽快由牙科或医疗专业人员检查；不要在输出中推断病因。

## Required Inputs

只使用用户明确提供的数据，并记录缺失项：

- 记录日期和数据来源（用户记录、牙医报告、设备或其他）。
- 症状原文、持续时间、变化和影响；不要改写成诊断名称。
- 刷牙、牙线、含氟产品、饮食和吸烟等习惯记录。
- 既往牙科就诊、专业建议和用户希望追踪的目标。
- 用户同意纳入的相关健康或用药信息。

不要要求不必要的身份信息。若输入来自不同来源，分别标注，不把估计值与临床测量值混合。

## Workflow

1. **确认范围**：复述用户希望比较的时间段、指标和输出形式。
2. **检查数据质量**：标记缺失日期、单位变化、重复记录和来源差异。
3. **建立时间线**：按日期排列症状、习惯变化、就诊和专业处理记录。
4. **描述趋势**：只报告可由记录支持的增加、减少、稳定或数据不足。
5. **检查升级条件**：先应用安全边界；存在紧急或持续问题时优先建议专业评估。
6. **关联但不归因**：可以指出事件在时间上同时发生，但不得声称因果关系。
7. **准备就诊材料**：生成简短摘要、缺失信息和可向牙医提出的问题。
8. **校验输出**：确认没有诊断、治疗、虚构数据或未经支持的风险百分比。

## Output Format

```markdown
## 记录范围
- 时间段：
- 数据来源：
- 缺失或不确定信息：

## 观察到的趋势
- 仅列出记录直接支持的变化。

## 需要及时处理的问题
- 紧急升级：有 / 无 / 信息不足
- 建议联系的专业人员及原因：

## 牙科就诊准备
- 一页摘要：
- 建议携带的记录：
- 可向牙医提出的问题：
```

## Evidence and Uncertainty Rules

- 将“用户报告”“设备记录”和“牙科专业人员记录”明确区分。
- 不把相关性表述为因果关系。
- 不创建未提供的检查结果、牙位、测量值或病史。
- 当记录不足时明确写“信息不足”，而不是给出低风险结论。
- 任何个体化医疗判断均交由合格的牙科或医疗专业人员完成。

## Limitations

- 本技能不能替代牙科检查、影像、病理或其他临床评估。
- 不根据照片或文字记录排除疾病。
- 不提供诊断、分期、治疗、处方或个体化用药建议。
- 如果用户需要临床结论，明确建议联系合格的牙科或医疗专业人员。
