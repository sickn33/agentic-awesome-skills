# 营养数据趋势整理

## When to Use

- 用户希望汇总一段时间的饮食记录、餐次、食物种类或已知营养标签数据。
- 用户需要比较记录期之间的食物模式，或为注册营养师/医生准备摘要。
- 任务不要求诊断营养缺乏、解释化验结果或决定补充剂剂量。

## Safety Boundary（安全边界）

- 不诊断缺乏症、进食障碍、过敏或其他疾病。
- 不设定个人化验目标，不解释异常化验是否需要治疗。
- 不推荐补充剂品牌、剂量、频率、疗程或与药物搭配；交由合格临床人员审查禁忌和相互作用。
- 不替代孕期、儿童、肾病、肝病、糖尿病、肿瘤治疗、吞咽困难或进食障碍的专业照护。
- 不把自制综合分数称为健康等级或医学风险。

如出现严重过敏反应、意识改变、无法进食饮水、持续呕吐、明显脱水或其他急性危险信号，停止分析并建议立即寻求当地急救或医疗帮助。

## 工作流程

1. **确认目标与范围**：记录时间段、记录方式、用户目标和已知专业治疗限制。
2. **检查来源**：区分包装标签、官方食物数据库、估算值和用户回忆；记录份量单位和缺失值。
3. **标准化记录**：统一日期、餐次和单位，但不猜测未知份量或配方。
4. **描述食物模式**：汇总食物种类、餐次规律和用户明确提供的营养值。只描述变化，不把相关性写成因果。
5. **使用外部基准时注明适用性**：仅引用当前官方、与年龄/地区/状态匹配的公开基准；不把群体参考值变成个人处方。
6. **提供低风险食物层面选项**：仅在不与用户限制冲突时，提出可回滚的一般选项，例如增加食物多样性或核对标签，并明确需要专业复核的条件。
7. **生成复核材料**：列出数据缺口、化验或补充剂问题、药物/疾病相关注意事项和给专业人员的问题。

```markdown
|---|---|---|

```

## Limitations

- 食物记录常受份量估算、遗漏和数据库差异影响。
- 群体参考值不能单独决定个人补充剂或治疗。
- 涉及化验异常、症状、疾病、药物或限制性饮食时，由注册营养师或医生作最终判断。
