# Mixpanel Insights

## Purpose

Gathers product usage insights from Mixpanel to identify early warning signs (drop-offs, pipeline failures) and highlight successes (high engagement, leading metrics).

## When to Use

Automatically activates when requested to:

- Run a Mixpanel health check
- Check the current product usage situation
- Identify least active companies in the last 7 days
- Look for anomaly signals or drop-offs

## How it Works

1. **Execute the Python Script:**
   Run the companion Python script located at `scripts/mixpanel_health_check.py` from the project root. This script fetches the last 14 days of data for core events (`file_finish`, `data_update_finish`, `table_get_record`) and aggregates it by `company_id` and date.
   *Command:* `python .claude/skills/mixpanel-insights/scripts/mixpanel_health_check.py`

2. **Read the Output:**
   Read the resulting JSON summary generated by the script (`insights_summary.json`).

3. **Analyze & Generate Report:**
   Analyze the JSON data to generate an insights report containing:
   - **Worst Signals:** Sharp drops in `data_update_finish` or `file_finish` week-over-week.
   - **Least Active:** Companies with the lowest or zero consumption (`table_get_record`) in the last 7 days.
   - **Good News:** Strong upward trends in pipeline completion or new active companies.
   Present the report to the user in clean Markdown. Include a tabular summary if helpful.

## Agentic Fallback

If the user asks for a very specific deep-dive (e.g., "Deep dive into exactly why Company X dropped off on Tuesday") and the standard script does not provide enough granularity, you may bypass the script and write ad-hoc Python to hit the Mixpanel API directly for that specific investigation. Should the question is related to other events than what the script provides, you can read the /context/ folders to find the list of all events and their usage patterns.
