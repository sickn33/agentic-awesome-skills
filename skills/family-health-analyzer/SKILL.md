# 家庭健康分析技能

## When to Use
- 需要分析家族病史、遗传风险或家庭层面的健康模式时使用。
- 任务涉及家庭健康报告、家族聚集性疾病识别或预防建议生成。
- 需要把多个家庭成员的健康数据汇总后做趋势或风险评估。

## 技能概述

本技能提供家庭健康数据的深度分析,包括:
- 家族病史模式整理
- 家族疾病模式识别
- 家庭共同问题分析
- 供临床咨询使用的问题清单
- 可视化报告生成

## 触发条件

当用户请求以下内容时,使用此技能:
- "家庭健康报告"
- "家族病史分析"
- "遗传风险评估"
- "家庭健康趋势"
- 执行 `/family report` 命令
- 执行 `/family risk` 命令

## 分析步骤

### 步骤1: 确定分析目标

识别用户请求类型:
- 家族病史分析
- 遗传风险评估
- 家庭健康趋势
- 家庭健康报告

### 步骤2: 读取家庭数据

**数据源:**
1. 主数据文件: `data/family-health-tracker.json`
2. 集成模块数据:
   - `data/hypertension-tracker.json`
   - `data/diabetes-tracker.json`
   - `data/profile.json`

### 步骤3: 数据验证与清洗

**验证项目:**
- 关系完整性
- 年龄合理性
- 数据一致性

### 步骤4: 遗传模式识别

**识别算法:**
1. 家族聚集性分析
2. 遗传模式识别
3. 记录相对较早发病的病例，并保留具体年龄；是否属于“早发”由疾病类型和当前临床指南决定

### 步骤5: 描述性模式与不确定性

不得根据亲属数量或发病年龄生成未经验证的百分比、权重或“高/中/低”遗传风险等级。仅描述观察到的模式，例如受影响的亲属关系、疾病类型、确诊年龄、多代聚集情况和缺失数据。明确区分已确认诊断、家属回忆和未知信息。

如家族史显示多名近亲患有相同或相关疾病、异常早发、罕见疾病或跨代聚集，应建议用户把整理后的家族史交给医生评估；涉及遗传检测或遗传性疾病解释时，转介合格的遗传咨询师或相关专科。不得推断遗传方式、个体患病概率或检测结论。

### 步骤6: 生成预防建议

**建议分类:**
- 资料完善:补充亲属关系、确诊名称、确诊年龄和报告来源
- 通用健康信息:仅提供不依赖个体诊断的低风险信息
- 专业咨询:列出应向医生或遗传咨询师询问的问题

**示例:**
```json
{
  "category": "clinical_discussion",
  "observation": "家族史中有多名近亲报告同类疾病",
  "uncertainty": "部分诊断和发病年龄尚未核实",
  "next_step": "携带家族史记录咨询医生或遗传咨询师，由其依据个人情况和当前指南决定是否筛查"
}
```

### 步骤7: 生成可视化报告

**HTML报告组件:**
1. 家谱树(ECharts树图)
2. 家族病史分布图（不显示未经验证的风险百分比）
3. 疾病分布饼图
4. 预防建议时间线

### 步骤8: 输出结果

**输出格式:**
1. 文本报告(简洁版):命令行输出
2. HTML报告(完整版):可视化图表

## 安全原则

### 医学安全边界
- ✅ 仅基于家族病史进行统计分析
- ✅ 整理可供医生判断的家族史模式和问题
- ✅ 明确标注不确定性
- ❌ 不进行遗传疾病诊断
- ❌ 不预测个体发病概率
- ❌ 不推荐具体治疗方案
- ❌ 不自行规定筛查起始年龄、频率或遗传检测

### 免责声明
每次分析输出必须包含:
```
⚠️ 免责声明:
1. 本分析基于家族病史统计,仅供参考
2. 遗传风险评估不预测个体发病
3. 所有医疗决策请咨询专业医师
4. 遗传咨询建议咨询专业遗传咨询师
```
